# Answer Formatting Bug PRD

## Problem

Final answers can render with section labels embedded in the wrong Markdown block:

- `What Supports This` can appear as a row inside the comparison table.
- `Caveats` can appear inline after the final support bullet instead of as its own section.
- `Evidence Used` can appear as one long inline string instead of a Markdown table.

This makes otherwise correct analysis look unpolished and difficult to scan.

## Root Cause

The answer is generated by an LLM, then normalized by the backend formatter. The formatter already handles many malformed cases, but the UI currently renders the answer string directly. If the answer is stale, already has numeric citations, or contains an inline model-generated evidence table, the UI may not rebuild section boundaries and evidence-table spacing before calling `st.markdown`.

Markdown tables are especially sensitive to missing blank lines. A section label immediately after a table can be interpreted as another table row.

## Goals

- Always render these sections as standalone Markdown headers:
  - `Bottom Line`
  - `Comparison Snapshot` or `Key Takeaways`
  - `What Supports This`
  - `Caveats`
  - `Evidence Used`
- Ensure support points appear under `What Supports This`.
- Ensure caveat points appear under `Caveats`.
- Ensure evidence appears under `Evidence Used` as a Markdown table.
- Preserve the existing numbered citation style.
- Keep answers concise and analyst-readable.

## Non-Goals

- Do not redesign the answer UI.
- Do not add a custom rich-text renderer.
- Do not change the underlying research/routing behavior.

## Acceptance Criteria

- A model answer containing `| What Supports This | | |` no longer renders that label as a table row.
- A model answer containing `. Caveats` splits `Caveats` into its own section.
- A model answer containing `Evidence Used | # | Evidence | ...` renders a proper evidence table.
- Already-numbered answers such as `[1][2]` keep numeric citations.
- Raw `SQL:` / `DOC:` citations are still converted to numeric citations.
- Existing tests continue to pass.

## Implementation Plan

1. Strengthen backend answer formatting for already-numbered answers.
2. Rebuild evidence tables from the evidence registry when numeric citations are present but raw labels are absent.
3. Defensively run UI answers through the formatter before `st.markdown`.
4. Add regression tests for the exact malformed pattern.
